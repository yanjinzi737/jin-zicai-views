# 建设计划

## 原则

分阶段推进，不一次性抓取大量材料；先解决来源、版权和结构，再扩充内容与自动化。

## 阶段 0：基础审计与骨架（本阶段）

- [x] 检查工作区已有文件与 Git 状态。
- [x] 分析参考仓库目录设计。
- [x] 检查参考仓库 LICENSE 与第三方材料边界。
- [x] 创建项目目录、初版 README、SKILL、AGENTS。
- [x] 创建 `docs/reference-audit.md`。
- [x] 运行基础检查。

停止条件：不收集金梓才语料，不实现抓取脚本，不形成投资方法结论。

## 阶段 1：来源地图与权利边界

- [ ] 以官方页面确认身份、任职机构和管理产品。
- [ ] 选少量代表性一手来源，建立来源优先级。
- [ ] 设计来源字段与材料收录规则。
- [ ] 决定原创内容许可证及第三方材料声明。

完成标准：形成来源白名单，不批量下载。

## 阶段 2：最小可用语料

- [ ] 选少量跨时期材料做人工核验样本。
- [ ] 建立语料索引和稳定命名规则。
- [ ] 验证引用、日期与人物归属。
- [ ] 根据样本修订 `SKILL.md`。

完成标准：能回答少量问题，也能可靠地说“不知道”。

## 阶段 3：方法框架与检索

- [ ] 从已核验语料归纳方法并绑定证据。
- [ ] 实现最小检索与索引校验脚本。
- [ ] 加入观点演变、冲突证据处理和代表性测试。

## 阶段 4：数据与言行对照（可选）

- [ ] 评估数据来源、许可与维护成本。
- [ ] 明确任职区间、报告期及经理归属规则。
- [ ] 仅在权利边界与校验机制成熟后接入。
